# 2026-07-08

## 项目分析：大道总纲（段永平投资问答录）website_duan

完成对 website_duan 项目的整体分析（Nuxt 4 + Vue 3 + Nuxt Content v3 静态站，Cloudflare Pages 部署）。核心结论：

### 结构
- 内容层 `content/dao/**/*.md` 共 121 篇，6 个 section（concepts 45 / company-people 27 / speeches 28 / qanda 6 / business-logic 8 / investment-logic 7）。
- zod schema 强制 frontmatter 5 必填字段，slug 全局唯一，交叉引用用 `/<slug>`。
- 仅 2 个路由页面 `index.vue` + `[slug].vue`；侧栏 LibrarySidebar 带搜索/折叠/自动定位；Twikoo 评论走独立 Worker。
- SEO 完整（JSON-LD/sitemap/robots/llms.txt/og-image/百度统计）；电子书由 generate-books.mjs 出 epub+pdf。

### 发现的关键问题（待处理）
1. **电子书漏章节**：`scripts/generate-books.mjs` 的 SECTIONS 只含 5 个 section，缺 `company-people`（27 篇，约 22%），epub/pdf 与该网站内容不同步。
2. **三套排序逻辑分散**：`index.vue`/`default.vue` 的 `daoCategoryOrder` 与 `generate-books.mjs` 的 SECTIONS 各自独立，新增分类需同步三处；`推荐书单` 在 index 为死配置。
3. 京东联盟链接 + QR 硬编码在 `index.vue`；`package.json` name 仍为 `mental-model`；`[slug].vue` 中 `isQanda` 的 `path.includes('/qanda/')` 分支基本不命中（靠 category 兜底）。
4. 已知本地 `generate` 易 OOM，文档已给 `NODE_OPTIONS=--max-old-space-size=8192` 解法。

### 建议优先级
- P0：补电子书 SECTIONS 加入 company-people。
- P1：把 daoCategoryOrder、京东链接、书单数据抽到 app/config/*.ts 单一数据源。
- P2：清理 package.json name、isQanda 无效分支。
