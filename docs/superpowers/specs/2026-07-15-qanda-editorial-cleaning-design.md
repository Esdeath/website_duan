# 投资问答录编辑性清洗设计

## 目标

对“投资问答录”现有 45 篇主题文章进行保守型编辑清洗，使排版整齐、符号规范，并删除确无信息价值的内容。

清洗不改动段永平原文的核心观点、论证、日期、金额、比例、估值、公司数据和口语风格。无法从上下文唯一确定的文字不修改。

## 内容范围

- 12 篇“投资原则与方法”。
- 12 篇“商业模式与经营”。
- 15 篇“公司案例”。
- 6 篇“人生与成长”。
- 固定的 20 篇原始来源、主题目录、迁移导读和清洗审计清单。

## 清洗规则

### 符号与排版

- 删除 `*()*`、孤立下划线、空括号和残留的 Markdown 装饰符号。
- 统一明显错误的重复问号、重复感叹号、重复句号及中英文标点混用。
- 修复连续逗号、句号前多余逗号、重复冒号等确定性排版错误。
- 保留原文有语气作用的单个问号、感叹号和省略号。
- 问答标识统一为“网友：”“段永平：”或“大道：”；不擅自替换有实际身份信息的署名。
- 问答和自然段之间保留一个空行，列表、引用及数据表保持原有结构。

### 错别字

- 仅修复能从上下文唯一确定的错别字或 OCR 错误。
- 修正通过显式替换表管理，每一条规则必须有原词、修正词和测试样例。
- 公司名、人名、英文术语或专业词无法确定时保留原文。
- 疑似原文事实错误不修改；必要时保留“原文如此”说明。

### 无意义内容

允许删除：

- 只有感谢、寒暄、点赞或表情，没有观点和事实的独立问答。
- 只有“呵呵”“哈哈”“不客气”等反应，且没有上下文信息的独立问答。
- 纯乱码、键盘误触且没有可恢复语义的内容。
- 与主题正文无关的超长网友名单或编辑性致谢名单。

必须保留：

- “不知道”“没研究过”“看不懂”等明确表达能力圈或判断边界的回答。
- 含观点、推理、案例、日期或数据的短回答。
- 虽为口语或玩笑，但对理解人物态度、企业文化或投资方法有作用的内容。
- 提问中的必要背景、附属列表和原始数据。

## 处理流程

### 来源与生成

1. 继续以审计清单固定的 20 篇原始文章为唯一来源。
2. 从原始来源提取完整问答单元，不直接把手工修正只写入生成文章。
3. 在生成器中依次执行：
   - 旧提问编号和 Markdown 残留清理；
   - 标点、空白和问答标识规范化；
   - 确定性错别字替换；
   - 无信息问答过滤。
4. 重新生成 45 篇主题文章、主题目录、20 篇迁移导读及清洗清单。

### 审计记录

每一项内容变化记录：

- 来源文章和问答单元 ID；
- 目标主题文章；
- 变化类型；
- 原始文本；
- 修正后文本，或删除原因。

变化类型至少包括：

- `format-normalized`：排版或符号修复；
- `typo-corrected`：确定性错别字修复；
- `discarded-no-information`：纯寒暄或无信息内容；
- `discarded-garbled`：无法恢复语义的乱码；
- `kept-original`：疑似问题但无法确定，保留原文。

### 人工复核

- 含金额、比例、年份、估值、公司名称、人名和英文术语的候选必须复核。
- 任何可能改变观点或事实含义的候选不得自动修正。
- 无法确认的候选保留，并在审计中标记为 `kept-original`。

## 实现边界

- 不使用通用拼写纠错器直接改写全文。
- 不重写段永平或网友的句式和语气。
- 不补充联网资料，不根据外部事实校正原文。
- 不重新调整已经确认的 45 篇主题与 15 篇公司案例结构。
- 不因篇幅长短删除有效问答。
- 本轮只做本地修改和验证，不自动提交实施改动、不推送、不部署。

## 验收标准

- 45 篇主题文章中不再出现已知 Markdown 残留、空括号、无效提问编号和确定性的重复错误标点。
- 问答角色、日期、金额、比例、估值及公司数据保持不变。
- 所有删除项都有明确且可审计的删除原因。
- 所有错别字修正规则都有回归测试。
- 0 个未分类问答。
- 0 个跨文章重复问答。
- 0 个未解释删除项。
- 0 个危险数据改动。
- 通过以下验证：
  - `node --test tests/*.test.mjs`
  - `node scripts/audit-qanda-cleaning.mjs`
  - `npm run typecheck`
  - `SKIP_DEPLOY=1 NODE_OPTIONS=--max-old-space-size=8192 npm run generate`
  - `npm run book`
  - `git diff --check`

## 风险控制

- 机械清理规则采用严格上下文匹配，避免删除正文中的有效符号。
- 错别字修正使用白名单，避免把原文用词改成新的观点。
- 删除规则以完整问答单元为边界，不从有效回答中截取句子。
- 生成前后对数字、日期、比例和英文标识做差异审计；不一致即视为错误并停止生成。
- 生成结果必须能从审计清单追溯到固定来源。
