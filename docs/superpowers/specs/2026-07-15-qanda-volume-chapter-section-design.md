# 投资问答录“卷—章—节”重组设计

## 背景

现有“投资问答录”已经完成跨文章清洗和主题归并，但实施时又按 5600 字目标把主题机械拆成大量 `-part-N` 文件。当前约 59 个主题被展开为约 230 篇页面，部分主题出现“1/16”“1/21”“1/18”等连续分篇。

这种结构保留了内容，却使目录层级过平、文章数量失真、连续阅读频繁中断。部分分篇还在同一个二级标题中间截断，读者无法从标题判断每个 part 的独立主题。

本次重组取消文章字数限制，保留已经确认的 59 个主题，将其恢复为 59 篇完整长文，并用“卷—章—节”表达层级。

## 目标

- 将约 230 个主题分篇合并为 59 篇完整章节。
- 建立“四卷、五十九章、章内多节”的稳定阅读结构。
- 不改变已经清洗保留的原始观点、问答、日期、案例和数据。
- 取消正文 5600/6000 字限制，不再按长度自动分篇。
- 让首页、侧边栏、主题目录、站内搜索和电子书使用同一套卷章顺序。
- 保持旧 URL 可访问，避免已发布的 `-part-N` 链接失效。

## 非目标

- 不联网补充新问答或校正原文事实。
- 不把问答体改写成连续论述文章。
- 不重新扩大或缩小已经确认的 59 个主题范围。
- 不因长文章再次设置字符数、问答数或小节数上限。
- 不在本次重组中调整其他内容分类的目录结构。

## 文章与小节结构

每一章对应一篇 Markdown 正文。页面标题由卷章元数据生成“第 N 章 + 主题名”，正文不再放重复的一级标题。

章内结构统一为：

```text
第 N 章 主题名
├── 第一节 核心定义或总原则
├── 第二节 适用边界与判断方法
├── 第三节 具体做法与问答
├── 第四节 公司或现实案例
└── 第五节 后续更新
```

实际节数由内容决定，不要求每章具备五节。二级标题表示“节”，三级标题用于节内的案例、细分观点或时间阶段。节标题必须表达主题，不使用“第一部分”“续”“其他”等无法独立理解的名称。

节编号在最终正文中连续显示为“第一节、第二节……”。不因内容较长拆成新页面。文章页现有目录读取二级标题，因此可直接形成章内导航；三级标题保留在正文中，但不强制进入浮动目录，以避免目录过长。

## 内容合并规则

1. 以 59 个基础 slug 为合并边界，例如 `wenda-invest-07-part-1` 至 `wenda-invest-07-part-16` 合并为 `wenda-invest-07`。
2. part 必须按自然数排序，禁止按字符串顺序造成 `part-10` 排在 `part-2` 前。
3. 删除各 part 的 frontmatter 后再合并正文，保留所有问答、日期、列表、引用、数字和站内链接。
4. 分页边界重复的标题只保留一次；归一化后完全相同的问答单元只保留清洗映射中标记为 `kept` 的版本。
5. 相同标题下的连续内容合并为一个节。标题相同但语义不同的内容，改用更具体的非观点性标题区分。
6. 不因观点相近而擅自合并不同日期、不同数字或带有新增信息的问答。
7. 章内优先按“总原则、定义与边界、具体做法、案例、后续更新”排列；同层问答保持原有时间顺序。
8. 保持短段落。仅在原句结束处拆段，不改写段永平的回答，也不拼接出原文没有的新结论。
9. 更新 `investment-qanda-cleaning-map.json`：所有原先指向 part slug 的 `kept` 单元改为指向基础章 slug；既有 `duplicate` 和 `discarded-no-information` 判定不因合并而改变。

## Frontmatter 与排序

59 篇正文统一使用 `type: qanda-chapter`，并增加以下字段：

```yaml
type: "qanda-chapter"
volume: "投资原则与方法"
volumeOrder: 1
chapterOrder: 7
```

`title` 保留不带编号的主题名，避免调整顺序时污染 SEO 标题和链接文字。页面、目录和侧边栏根据 `chapterOrder` 显示中文章号。

所有章节继续使用：

- `category: "投资问答录"`
- 3–6 个 `tags`
- 全局唯一的基础 `slug`
- 完整的 `description / seoTitle / seoDescription`

`order` 继续承担全站和电子书排序：主题目录为 190；第一卷 200–219；第二卷 230–249；第三卷 260–309；第四卷 320–329。每卷内部按章序连续递增。

`content.config.ts` 增加 `volume / volumeOrder / chapterOrder` schema，三个字段仅对 `qanda-chapter` 必填，其他历史内容不受影响。

## 目录和阅读体验

### 主题总目录

`wenda-topic-index` 保留为主题入口，但从平铺 230 个 part 链接改为四卷结构。每卷列出章号、主题名和章内节标题；节标题链接到对应文章锚点。

目录展示示例：

```text
第一卷 投资原则与方法
  第七章 现金流折现、PE 与定性判断
    第一节 现金流折现的基本含义
    第二节 不要机械套用估值公式
    第三节 PE 与定性判断
```

### 侧边栏

“投资问答录”分类下先显示“主题总目录”，再显示四个可折叠卷。卷内只列 59 个章标题，不列节，也不显示旧迁移页。

当前文章所在的卷自动展开，当前章保持高亮。搜索仍可检索完整正文；命中某章时返回该章，不把同一章拆成多个结果。

### 文章页

文章页标题上方显示卷名，标题显示章号和主题名。右侧或移动端文章目录展示二级标题，也就是本章各节，并继续支持锚点跳转和滚动高亮。

文章底部增加“上一章 / 返回本卷目录 / 下一章”导航。跨卷边界时，下一章进入下一卷第一章。

### 首页

首页“投资问答录”卡片显示“59 篇”，不把主题总目录和迁移页计入正文数量。点击卡片进入 `wenda-topic-index`，而不是直接进入第一章。

## URL 与旧链接兼容

新正文采用基础 slug：

- `/wenda-invest-07`
- `/wenda-business-08`
- `/wenda-company-apple-01`
- `/wenda-life-04`

现有 `-part-N` 内容文件在合并后删除。为这些已发布地址生成 Cloudflare Pages 静态重定向规则，统一 301 到对应基础 slug，不尝试跳到不稳定的小节锚点。

原清洗前的 20 个 URL 继续保留 `legacy-index` 迁移导读页，并把原先指向 part 的链接更新为新章节链接。

旧 part URL 和 20 个迁移页均不出现在首页、侧边栏、站内搜索、`llms-full.txt`、电子书或 sitemap 中。part URL 由重定向规则保留；20 个历史迁移页继续通过显式预渲染路由生成。

新章节使用自身 canonical URL。迁移导读页的 canonical 指向最相关的新章节或主题总目录。

## 电子书、LLM 与 sitemap

- 电子书继续按现有 `qanda / investment-logic / business-logic` 一级目录取材，但只收录 `qanda-chapter` 正文，跳过 `topic-index` 和 `legacy-index`。
- 电子书目录增加卷名层级，卷内显示章号和主题名；章内二级标题作为 EPUB/PDF 的节。
- `llms-full.txt` 收录 59 篇完整章节，跳过主题目录和所有迁移入口。
- sitemap 收录首页、主题总目录和 59 篇新章节；排除 `legacy-index`。
- 静态生成必须产出 59 个基础 slug 页面、主题总目录、20 个旧迁移页，以及 part 重定向规则。

## 生成与审计工具

现有 `scripts/generate-qanda-topics.mjs` 从“按字符分篇生成器”调整为“按主题生成章节”：

- 删除目标 5600 字和硬上限 6000 字逻辑。
- 每个主题始终只输出一个基础 slug 文件。
- 生成连续的卷章元数据和节编号。
- 生成主题目录、part 重定向清单和旧迁移页链接。
- 重复运行必须得到相同文件内容和顺序。

`scripts/audit-qanda-cleaning.mjs` 调整为只读检查：

- 正文恰好 59 篇，四卷数量分别为 12、12、29、6。
- `volumeOrder / chapterOrder` 连续且不重复。
- 不存在 `qanda-chapter` 的 `-part-N` slug。
- 每章至少有一个“第 N 节”二级标题，节编号连续。
- 59 篇正文之间不存在归一化后完全相同、长度不少于 50 字的问答单元。
- 清洗映射中的所有 `kept` 单元指向有效基础 slug。
- 所有旧 part slug 都具有有效重定向目标。
- 章节没有字数上限检查，但继续检查异常长段落和必填 frontmatter。

## 测试与验收

执行：

```bash
node scripts/generate-qanda-topics.mjs
node scripts/audit-qanda-cleaning.mjs
npm run typecheck
SKIP_DEPLOY=1 NODE_OPTIONS=--max-old-space-size=8192 npm run generate
npm run book
git diff --check
```

验收标准：

1. 站内只展示 59 篇投资问答录正文，数量分布为 12、12、29、6。
2. 主题总目录和侧边栏均按四卷、五十九章展示，不再出现 `1/16` 等 part 标题。
3. 任意章节都能通过二级标题目录跳转到各节，并能使用上一章、下一章连续阅读。
4. 合并前后的 `kept` 问答单元数量一致；抽查金额、比例、年份、公司数据、说话人和日期均未变化。
5. 旧 `-part-N` 地址重定向到对应基础章节；20 个原始 URL 仍可打开迁移导读。
6. 首页计数为 59；搜索每个主题只返回一个章节结果。
7. sitemap、`llms-full.txt` 和电子书不包含迁移入口或 part 页面。
8. 类型检查、静态生成、电子书生成、清洗审计和 `git diff --check` 全部通过。

## 实施边界

本设计的实施只修改本地仓库并进行本地验证。除非用户另行明确要求，不部署、不推送。正文重组完成后再根据用户指示决定是否提交和发布。
