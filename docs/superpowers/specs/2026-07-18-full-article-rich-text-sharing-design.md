# 全文富文本分享设计

## 目标

文章页只显示一个“分享”按钮。读者点击后，浏览器把固定来源说明、原文链接、文章标题和完整正文写入剪贴板。富文本内容保留标题层级、段落、列表、引用和粗体。正文链接只保留显示文字，不保留跳转地址。

## 分享内容

剪贴板内容按以下顺序排列：

1. 固定来源“段永平投资问答录”和可点击的完整文章 URL。
2. 一次文章标题。
3. 文章正文。
4. 再次显示固定来源和可点击的完整文章 URL。

正文沿用页面已经渲染的 HTML，保留标题层级、段落、列表、引用和粗体。分享组件把正文中的链接拆成普通文字；首尾原文链接仍可点击。分享组件移除克隆正文开头自带的一级标题，再由统一格式插入一次标题。来源说明只使用固定文案，不读取文章 `source` 元数据。分享内容不包含摘要、标签、侧栏目录、文章来源、章节导航和评论区。

## 交互

- 删除“分享链接”和“分享图片”两个按钮，只保留一个“分享”按钮。
- 点击按钮后直接复制，不打开预览弹窗。
- 复制期间禁用按钮并显示“复制中...”。成功后短暂显示“已复制”。
- 复制失败时显示“复制失败”，随后恢复为“分享”，允许读者重试。

## 实现

文章页给正文的 `ContentRenderer` 容器添加稳定的 DOM `id`，并把该 `id` 传给分享组件。分享组件克隆该容器，移除开头的一级标题，把正文锚点替换成其文字内容，再交给格式工具添加首尾来源、原文链接和单一标题。

支持 `ClipboardItem` 的浏览器同时写入 `text/html` 和 `text/plain`。富文本编辑器读取 HTML 并保留格式；纯文本输入框读取纯文本。浏览器不支持富文本剪贴板时，组件使用 `navigator.clipboard.writeText` 复制同等顺序的纯文本内容。

## 验证

- 单元测试覆盖分享内容顺序、正文纳入范围、页面附属内容排除和纯文本降级。
- 类型检查确认 Vue 与浏览器剪贴板类型正确。
- 静态生成确认文章页可预渲染，分享逻辑只在用户点击后访问浏览器 API。
