# 投资问答录逐篇清洗与逻辑重排设计

## 目标

对“投资问答录”现有 45 篇主题文章逐篇清洗。处理篇内及跨篇重复内容，删除确认无信息的内容，并在问题与答案始终绑定的前提下调整问答顺序，使每篇文章结构更合理、逻辑更连贯。

本轮保持现有文章体系不变：投资原则与方法 12 篇、商业模式与经营 12 篇、公司案例 15 篇、人生与成长 6 篇。不重新拆分或合并文章，不限制单篇字数。

## 内容来源与现状保护

- 继续使用清洗审计清单固定的 20 篇原始文章作为内容来源。
- 当前 45 篇成品文章中已经人工确认的删除、修正和问答补标必须转成可重复执行的生成规则。
- 特别保护 `f45ab92` 和 `912ce52` 对 `wenda-invest-01`、`wenda-invest-02` 的人工整理结果，重新生成时不得恢复已确认删除的内容。
- 原文的核心观点、日期、金额、比例、估值、公司数据和有效案例不得改写。

## 问答单元与问答组

### 问答单元

每个问答单元必须完整包含：

- 提问及理解该问题所需的背景；
- 段永平对应的回答；
- 对应日期；
- 与该问答直接相关的列表、引用和数据。

排序和去重只能以完整问答单元为最小边界。不得单独移动问题或回答，不得把不同问答的内容拼接成新的回答。

### 问答组

相邻的追问、补充回答或依赖上文才能理解的连续对话组成问答组。问答组整体移动，内部保持原始顺序。

以下情况优先视为同一问答组：

- 后续问题使用“这个”“上述”“那为什么”等指代上文的表达；
- 同一用户围绕同一结论连续追问；
- 回答明确补充或修正紧邻的前一回答；
- 多轮对话共享同一个日期、案例或附属数据。

无法确定是否依赖上下文时，保留相邻关系，不拆分。

## 去重与删除规则

### 重复内容

- 先做篇内精确去重，再做跨篇精确去重和近似重复检查。
- 同一问答只保留一个信息最完整的主本。
- 主本优先选择具有完整问题、明确回答、日期、较完整背景和更深主题上下文的版本。
- 包含不同日期、数字、案例或新增观点的相似问答不得视为重复。
- 近似重复候选必须明确记录为保留或删除，不允许留下未处理状态。

### 无信息内容

允许删除：

- 空回答；
- 只有感谢、寒暄、点赞、表情或无上下文反应的独立问答；
- 无法恢复语义的乱码；
- 与主题正文无关的编辑说明、贡献者名单或残留占位内容；
- 已确认重复的副本。

必须保留：

- “不知道”“没研究过”“看不懂”等能力圈表达；
- 含观点、推理、事实、案例、日期或数据的短回答；
- 虽然口语化，但能体现态度、判断边界或企业文化的回答；
- 理解回答所必需的提问背景、列表和数据。

## 篇内结构与排序

每篇文章按以下逻辑层级组织：

1. 核心原则与结论；
2. 概念定义与适用范围；
3. 判断边界与常见误区；
4. 具体方法和操作原则；
5. 案例、公司或场景分析；
6. 后续补充与观点更新。

实际排序键为：

1. 主题小节顺序；
2. 逻辑层级；
3. 日期；
4. 原始来源顺序。

具体规则：

- 先按文章现有主题小节归类，不让内容跨入不相关小节。
- 同一小节内按逻辑层级排序。
- 同一层级内按日期从早到晚排列，以呈现观点演变。
- 无日期问答保持原始相对顺序，并放在最接近的同类内容旁边。
- 公司合并文章继续保留公司三级标题，先按公司排列，再在公司内部执行逻辑排序。
- 问答组使用组内最早明确日期参与排序；组内顺序不变。
- 长回答只在原句结束处拆段，不拆断列表、引用或数据。

## 实现架构

### 单元识别层

扩展现有问答解析逻辑，为每个单元保留稳定 ID、来源文章、标题路径、原始顺序、日期和说话人信息，并建立问答完整性检查。

### 问答组识别层

在单元解析之后、去重之前识别连续对话。问答组具有稳定组 ID、成员单元 ID、组内顺序和排序日期。

### 清洗决策层

继续使用现有精确去重、近似重复审核、无信息分类和编辑清洗模块，并新增人工决策清单：

- 固定已确认删除的单元；
- 固定必须保留的能力圈表达和短回答；
- 固定近似重复的主本；
- 固定人工整理产生的修正。

### 逻辑排序层

为保留的问答单元或问答组分配：

- 目标文章；
- 主题小节；
- 逻辑层级；
- 日期排序值；
- 原始顺序兜底值。

排序优先使用主题配置和明确规则。无法可靠归类的内容保持原小节及原始相对顺序，不使用生成式改写猜测其位置。

### 生成与审计层

重新生成 45 篇成品文章、主题目录、20 篇迁移导读及清洗审计清单。生成器必须可重复执行，相同输入产生相同输出。

## 审计记录

审计清单需要记录：

- 每篇文章清洗前后的问答数量；
- 每个来源单元的保留、重复或删除状态；
- 重复内容的主本及目标文章；
- 每个问答组的成员和组内顺序；
- 问答或问答组调整前后的文章、小节和排序位置；
- 每项删除的明确原因；
- 每项文字或排版修正的前后内容；
- 数字、日期、金额、比例、估值及公司数据的危险变化检查；
- 45 篇文章各自的清洗统计。

## 风险控制

- 问题和答案完整性检查失败时停止生成。
- 问答组成员被分配到不同目标或不同小节时停止生成。
- 任意危险数字或日期变化时停止生成。
- 近似重复候选没有明确结论时审计失败。
- 当前人工删除内容重新出现时审计失败。
- 无法可靠归类的内容保持原顺序，不为追求形式整齐而强行移动。
- 不联网补写或校正原文事实，不改写问答观点。

## 验收标准

- 45 篇文章均生成逐篇清洗统计。
- 0 个问题与答案分离。
- 0 个问答组被拆散。
- 0 个未说明删除项。
- 0 个危险数字或日期变化。
- 0 个确认重复问答。
- 0 个未处理近似重复候选。
- 当前人工删除内容不会在重新生成后恢复。
- 文章内部符合“主题小节、逻辑层级、日期、原始顺序”的排序规则。
- 清洗审计、全部测试、类型检查、静态生成和电子书生成通过。
